# 小鹏公开 Demo 发布后自检清单

## 1. 适用范围

本清单只用于“面试官可直接访问的公开 demo”。

不适用于商业上线验收，不代表内部系统已打通。

---

## 2. 发布前检查

### 代码与构建

1. `frontend lint` 通过。
2. `frontend build` 通过。
3. `backend smoke` 通过。
4. `backend eval` 通过。

### 配置与环境

1. 前端 `NEXT_PUBLIC_API_URL` 指向线上后端地址。
2. 后端 `ALLOWED_ORIGINS` 已配置线上域名。
3. 后端 `OPS_ACCESS_TOKEN` 已配置，`/ops` 仅内部可看。
4. 存储与数据库配置符合当前部署方案（file 或 postgres）。
5. HTTPS 可用，证书有效。

### 对外边界

1. 首页可见“公开网页抓取快照 / 未接内部系统”说明。
2. 配置器、权益、门店页面均可见边界说明。
3. 试驾结果文案不误导为真实顾问已接单。
4. 页面有官方来源链接，且描述为“以官网/门店为准”。

---

## 3. 发布后 10 分钟冒烟自检

### 基础可用性

1. 打开首页不报错，静态资源加载正常。
2. 能发送一条推荐问题并返回结果。
3. 能发起一次车型对比。
4. 能进入配置器并完成一次配置摘要。
5. 能打开权益弹窗并看到快照条目。
6. 能打开门店弹窗并看到快照条目。
7. 能完成一次试驾 CTA 提交。

### 链路完整性

1. 主链路可在 2-3 分钟内走完：
   `首页 -> 推荐 -> 对比 -> 配置器 -> 试驾/门店 CTA`
2. 所有关键步骤无 5xx 错误。
3. 接口响应没有明显超时或卡死。

### 文案一致性

1. 没有“已接入内部系统”或类似暗示。
2. 没有“顾问将尽快联系你”这类真实业务承诺。
3. 所有“预约”相关文案均提示 demo 边界和官方入口。

---

## 4. /ops 最低检查项（内部）

1. `health` 正常。
2. `business-data` 状态可读，来源字段完整。
3. `crm outbox` 可读，状态流转正常（mock-first 预期）。
4. `audit log` 能看到访问记录。
5. 发布前后核心指标无异常飙升（错误率、延迟）。

---

## 5. 故障回退预案

出现以下问题时，暂停对外发送链接：

1. 主链路任一关键步骤不可用。
2. 页面文案出现“已接内部系统”误导。
3. 权益/门店快照无法加载且无 fallback。
4. 线上错误率显著升高或持续 5xx。

回退动作：

1. 先回滚到上一可用版本。
2. 重新执行本清单第 2、3 节。
3. 复盘并记录问题，再恢复外发。

---

## 6. 发布记录模板

建议每次发布记录一行：

1. 发布时间（UTC+8）
2. 版本号或 commit id
3. 发布人
4. 自检结果（通过/失败）
5. 发现问题与处理结论
