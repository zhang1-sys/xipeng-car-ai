# 零基础部署全流程（智选车 Agent）

按顺序做即可。你只要会：**复制粘贴命令**、**在网页里填 API Key**、**买一台能装 Docker 的云服务器（可选）**。

---

## 你需要提前准备的东西

| 物品 | 说明 |
|------|------|
| **Kimi（月之暗面）API Key** | [平台](https://platform.moonshot.cn/) 注册 → 创建 API Key；或改用 OpenAI 兼容接口（见 `backend/.env.example`） |
| **电脑已安装 Node.js 20+** | 本地先跑通再上线更稳：[下载](https://nodejs.org/) |
| **（上线）云服务器或 PaaS** | 二选一，见下文「方案 A」「方案 B」 |

## 第一步：在你自己电脑上先跑通（必做）

用来确认代码没问题，再部署。

### 1. 配置后端

```bash
cd backend
copy .env.example .env
```

用记事本打开 `backend\.env`，把 `MOONSHOT_API_KEY=` 后面改成你的真实 Key。保存。

建议同时补上：

- `OPS_ACCESS_TOKEN=一段长随机串`
- 若要切到数据库存储，再补 `STORAGE_PROVIDER=postgres`
- 若要切到向量检索，再补 `KNOWLEDGE_RETRIEVAL_PROVIDER=postgres`

```bash
npm install
npm start
```

看到 `API listening on http://localhost:3001` 即成功。窗口不要关。

如果你暂时还没配 API Key，聊天接口也能以**本地兜底模式**先跑起来，方便联调页面；但正式演示和上线请务必配置 Kimi / OpenAI 兼容模型。

### 2. 配置前端（新开一个终端）

```bash
cd frontend
copy .env.local.example .env.local
```

`.env.local` 里保持 `NEXT_PUBLIC_API_URL=http://localhost:3001` 即可。

```bash
npm install
npm run dev
```

浏览器打开：**http://localhost:3000** ，发一句话测试对话。

如果你本机 `next dev` 启动后页面资源异常，直接改用：

```bash
npm run build
npm start
```

当前仓库已经把 `standalone` 静态资源复制逻辑封装进 `npm start`，本地预览更稳。

## 方案 A：一台云服务器 + Docker（推荐，适合「一个网址搞定」）

适合：阿里云 / 腾讯云 / 华为云等 **Linux 云主机**（建议 2 核 4G，系统 Ubuntu 22.04）。

### A1. 服务器上安装 Docker

登录服务器 SSH 后执行（Ubuntu 示例）：

```bash
sudo apt update
sudo apt install -y ca-certificates curl
sudo install -m 0755 -d /etc/apt/keyrings
sudo curl -fsSL https://download.docker.com/linux/ubuntu/gpg -o /etc/apt/keyrings/docker.asc
sudo chmod a+r /etc/apt/keyrings/docker.asc
echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.asc] https://download.docker.com/linux/ubuntu $(. /etc/os-release && echo "$VERSION_CODENAME") stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
sudo apt update
sudo apt install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin
```

把项目上传到服务器（任选一种）：

- **Git**：服务器上 `git clone` 你的仓库；
- **SFTP / 控制台上传**：把整个 `xpeng-car-ai` 文件夹传到服务器，例如 `/home/ubuntu/xpeng-car-ai`。

### A2. 配置环境变量

在服务器上：

```bash
cd /home/ubuntu/xpeng-car-ai/backend
cp .env.example .env
nano .env
```

填好 `MOONSHOT_API_KEY` 等，保存。

如果你准备把内部运营页一起上线，建议同时在 `backend/.env` 里加：

```bash
OPS_ACCESS_TOKEN=请改成一段长随机串
```

如果你准备直接跑 Postgres + pgvector，也建议一并补：

```bash
STORAGE_PROVIDER=postgres
KNOWLEDGE_RETRIEVAL_PROVIDER=postgres
DATABASE_URL=postgresql://xpeng:xpeng_dev_password@db:5432/xpeng_car_ai
DATABASE_SSL=false
```

如果你已经有业务侧提供的 JSON 数据接口，也可以继续补：

```bash
CATALOG_SOURCE_URL=https://你的数据源/catalog.json
STORES_SOURCE_URL=https://你的数据源/stores.json
RIGHTS_SOURCE_URL=https://你的数据源/rights.json
ADVISOR_SOURCE_URL=https://你的数据源/advisors.json
```

后端会在启动时先拉一次，之后定时刷新；如果远程失败，会自动回退到仓库里的本地文件，不会直接把服务打挂。

```bash
cd /home/ubuntu/xpeng-car-ai/deploy
cp env.example .env
nano .env
```

把 `PUBLIC_ORIGIN` 改成用户访问时**在浏览器地址栏里看到的前缀**（不要末尾 `/`）：

- 暂时用 IP：`http://你的公网IP`
- 若 80 端口改成 8080，则要写：`http://你的公网IP:8080`（与 `HTTP_PORT` 一致）

### A3. 一键启动

```bash
cd /home/ubuntu/xpeng-car-ai/deploy
docker compose up -d --build
```

第一次会较久（下载镜像 + 构建）。完成后：

- 浏览器访问：`http://你的公网IP`（或你设的端口）
- 应看到**带样式的**页面，且能聊天。

### A4. 常用命令

```bash
cd /home/ubuntu/xpeng-car-ai/deploy
docker compose logs -f          # 看日志
docker compose down             # 停止
docker compose up -d --build    # 改代码或 .env 后重新构建启动
```

**改了 `PUBLIC_ORIGIN` 或前端代码**：必须重新 `docker compose up -d --build`（前端 Key 在构建时写入）。

**只改后端 `.env`（例如 Key）**：`docker compose up -d` 即可重启加载。

### A5. 域名与 HTTPS（上线建议）

1. 域名解析到服务器公网 IP。  
2. `deploy/.env` 里 `PUBLIC_ORIGIN=https://你的域名`（先确保 443 证书，可用 **Caddy** 或 **Certbot + Nginx** 自动证书；超出本文时可搜索「nginx letsencrypt」）。  
3. 重新 `docker compose up -d --build`。

## 方案 B：不租服务器——Vercel（前端）+ Railway（后端）

全程网页点选，适合不想管 Linux 的用户。

### B1. 把项目放到 GitHub

1. 在 [GitHub](https://github.com) 新建空仓库。  
2. 在你电脑项目根目录 `xpeng-car-ai` 里：

```bash
git init
git add .
git commit -m "init"
git remote add origin https://github.com/你的用户名/你的仓库.git
git branch -M main
git push -u origin main
```

（若已有仓库，直接 push。）

### B2. 部署后端（Railway 示例）

1. 打开 [railway.app](https://railway.app) 注册并登录。  
2. **New Project** → **Deploy from GitHub repo** → 选你的仓库。  
3. 在项目里 **Add Service** 或进入该服务 → **Settings**：  
   - **Root Directory** 填：`backend`  
   - **Start Command**：`npm start`  
4. **Variables** 里添加与 `backend/.env.example` 相同的变量（至少 `MOONSHOT_API_KEY`、`MOONSHOT_BASE_URL`、`MOONSHOT_MODEL`，`PORT` 一般由平台注入，可不填或填 `PORT`）。  
5. 部署完成后，Railway 会给一个公网 URL，例如 `https://xxx.up.railway.app`。复制下来，称为 **API 地址**。

在浏览器打开：`你的API地址/health` ，若返回 JSON 即后端正常。

### B3. 部署前端（Vercel）

1. 打开 [vercel.com](https://vercel.com) 用 GitHub 登录。  
2. **Add New Project** → 导入同一仓库。  
3. **Root Directory** 选：`frontend`  
4. **Environment Variables** 添加：  
   - `NEXT_PUBLIC_API_URL` = **你的 API 地址**（不要末尾 `/`，例如 `https://xxx.up.railway.app`）  
5. **Deploy**。

部署完成后用 Vercel 提供的 `https://xxx.vercel.app` 访问即可。

如果后端配置了 `OPS_ACCESS_TOKEN`，内部运营页在：

```text
https://你的前端域名/ops
```

首次进入时，把 Token 粘进去即可查看 readiness、审计、CRM outbox、知识库状态等内部数据。

### B4. 方案 B 的注意点

- 免费额度有限，访问量大了要升级套餐。  
- 试驾留资和会话记忆默认写在容器文件里；容器重启若未挂持久盘，**可能丢 `leads.jsonl` 与 `sessions.json`**。正式商用建议接 `LEAD_WEBHOOK_URL` 推到 CRM，并把会话记忆改成 Redis / Postgres。

| 检查项 | 做法 |
|--------|------|
| 页面有颜色、圆角、按钮不像系统默认灰块 | 若仍是纯文字，见主 `README.md`「页面像纯文字」一节（多为 `/_next/static` 未加载） |

## 遇到问题怎么办

1. **聊天报错 / 网络错误**：核对 `NEXT_PUBLIC_API_URL` 是否与线上一致、后端是否运行、浏览器 F12 → **Network** 里 `/api/chat` 是否红字。  
2. **Kimi 报错**：检查 Key、余额、`MOONSHOT_MODEL` 是否可用。  
3. **Docker 构建失败**：把终端完整报错复制下来，对照是否缺 `backend/.env`、`deploy/.env`。  
4. **试驾提交失败**：确认前端已勾选隐私授权说明；当前版本把授权作为必填字段。

更细的 API 说明、高德 Key、留资 Webhook 见仓库根目录 **README.md**。

如果你改了较多代码，建议在后端启动后先执行一次：

```bash
cd backend
npm run smoke
```

这样可以在上线前快速发现推荐、对比、多轮记忆和试驾链路的回归问题。
